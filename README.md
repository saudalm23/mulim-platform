## المشكلة

عند صدور تشريع جديد (مثال: لائحة تتضمن 120 متطلباً)، تحتاج البنوك وشركات التقنية المالية والتمويل والتأمين إلى **أسابيع من التحليل اليدوي** للإجابة على:
كم سياسة ستتغير؟ أي الإدارات ستتأثر؟ كم موظفاً؟ كم التكلفة؟ ما مستوى المخاطر؟ وهل تكفي المهلة النظامية؟

## الحل

مُلم يحسب **التقاطع** بين التشريع الجديد والوثائق الداخلية للمؤسسة عبر ثلاث مراحل:

| المرحلة | الوصف |
|---|---|
| 1️⃣ استخراج المتطلبات | تفكيك التشريع إلى متطلبات ذرّية مصنّفة (يجب / يحظر / يجوز / تعريفي) |
| 2️⃣ الربط الذكي | بحث هجين (دلالي + كلمات مفتاحية) + حَكَم لغوي يصنّف كل متطلب: مغطى / جزئي / فجوة، مع **درجة ثقة** |
| 3️⃣ حسبة الأثر | محرّك حتمي قابل للتفسير يحسب: درجة الأثر 0–100، التكلفة، الوقت، والإدارات والأنظمة والموظفين المتأثرين |

### نموذج 80/20 — الإنسان في الحلقة
- ثقة ≥ 85٪ ← اعتماد آلي (يُنجز ~80٪ من العمل)
- ثقة 50–85٪ ← قائمة مراجعة بشرية لموظف الالتزام (الـ 20٪ المتبقية)
- كل قرار موثّق ومرتبط بالمادة النظامية والبند الداخلي — **قابلية تدقيق كاملة**

## المميزات في العرض المباشر
- 📊 لوحة الأثر: درجة 0–100 + المتغيرات الثمانية (سياسات، إجراءات، إدارات، موظفون، أنظمة، خطورة، تكلفة، وقت)
- ⏱️ مؤشر جدوى المهلة النظامية (الوقت المطلوب مقابل المتاح)
- 🗺️ خريطة الفجوات: مصفوفة المتطلبات × السياسات
- 👤 قائمة المراجعة البشرية — القرارات تُحدّث درجة الأثر لحظياً
- 🎛️ محاكاة السيناريوهات (What-if): تأجيل الفجوات، تكلفة التدريب، طاقة الفريق
- 📋 خطة تنفيذ تلقائية مجمّعة حسب الإدارة
- 💬 اسأل التشريع: إجابات مستندة إلى نص المواد

## الفئة المستهدفة
البنوك · شركات التقنية المالية (261+ شركة مرخصة في المملكة) · شركات التمويل · شركات التأمين

## لماذا الآن؟ (السوق)
- سوق RegTech السعودي: من **317 مليون دولار (2023)** إلى **866 مليون دولار (2029)** بنمو سنوي ~16.7٪
- لا يوجد منافس **عربي أولاً** مبني على تشريعات ساما وهيئة السوق المالية والهيئة الوطنية للأمن السيبراني ونظام حماية البيانات الشخصية
- المنافسون العالميون (4CRisk، Regology بسعر يبدأ من 1,700 دولار/مستخدم/شهر) يتوقفون عند تحليل الفجوات — مُلم يقدّم **محاكاة الأثر**: التكلفة والوقت والموظفون والأنظمة
